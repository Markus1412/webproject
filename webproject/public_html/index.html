Why RuterFeedback?

Problem

Vandalism on public transport does not get the attention it needs.

There are plenty of vehicles that have obvious flaws, and if it got more attention from the public it would get sorted out faster.

We think the problem exists partly because notifying Ruter is a hassle, due to an old and outdated system.

Currently you need to call them to report a problem, and most people would rather not report it at all then.

It is hard to know how you can report a potential case of vandalism, and Ruter could do a slightly better job at letting people know how they can help.

Solution

Our proposed solution to the vandalism is to create an extension for the RuterReise application.

The main objective of this application is to get people to report vandalism, and it would be important for the application to be easy to use to avoid people getting too annoyed reporting a problem.

A user could use the feedback function during a ride or within 24 hours after getting off.

This application should make it easier to report cases of vandalism, especially for younger travelers.

Challenges & Benefits

Challenges

Since the replies will be available to see for everyone, there is a possibility for spam. To avoid this we would implement a GPS-restriction that would only allow a customer to give feedback if they are (or were) in the vehicle in the past 24 hours.

The application might be too technical for older people, but it would be a non-issue assuming they kept the option to call Ruter.

Benefits

People could automatically get involved/aware by getting a notification once the RuterReise app has been updated with the Ruterfeedback function. This would be a one-time thing, but it would make sure people are aware of the new possibilities within the app.

Having an app like this can generate different statistics that might be valuable for Ruter.

Because the app is included in the RuterReise app customers would not have to deal with extra clutter on their phones. Ideally the app-extension would be lightweight.

The app will work as a base platform for Ruter, and they could expand it easily.

The app could be expanded to include feedback on drivers/general routes. This could be hidden feedback to avoid spam and hopefully only generate proper responses.
